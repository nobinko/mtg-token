# MTG Token Finder

フィーチャー卓の配信準備で、バルクのトークン束から必要な現物を抜き出すためのローカルWebツールです。

---

## はじめて使う方へ

### 1. Node.js をインストールする（初回のみ）

https://nodejs.org/ja/ を開き、「LTS」と書かれたボタンでインストーラをダウンロードして実行してください。
途中の選択肢はすべてデフォルトのままで構いません。

### 2. `start.bat` をダブルクリックする（通常はこちら）

リポジトリのフォルダにある **`start.bat`** をダブルクリックすると

- 黒いウィンドウが開き、サーバが起動します
- 数秒後にブラウザが自動で開き、ツールが使えます

**初回のみ**「パッケージをインストールしています」と表示されて少し待ちます（次回からはすぐ起動します）。

### 3. ツールを使い終わったら

黒いウィンドウを閉じてください。サーバも自動で停止します。

### トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「Node.js が見つかりません」と表示される | https://nodejs.org/ja/ からインストールして再起動 |
| ブラウザが開かない | 手動で `http://localhost:5177` を開く |
| 「インストールに失敗しました」と表示される | ネット接続を確認して `start.bat` を再実行 |
| 黒いウィンドウが一瞬で閉じる | PowerShellでこのフォルダを開き、`.\start.bat` を実行して表示されたエラーを確認 |
| `npm install` が権限エラーになる | 会社PCやウイルス対策ソフトの制限があり得ます。PowerShellを開き直すか、必要なら管理者として実行 |

---

このツールは「カード検索」ではなく「現場の準備表」です。デッキリストを集め、トークン・紋章・物理的なコピー・裏向き補助など、プレイ中にカード状の現物として必要になりそうなものをエキスパンション順または種類別に並べます。

## PowerShellで起動する場合

```powershell
npm install
npm start
```

初回は `npm install` が必要です。2回目以降は `npm start` だけで起動できます。

起動後、ブラウザで開きます。

```text
http://localhost:5177
```

停止するときは、サーバを起動したPowerShellで `Ctrl+C` を押します。

## 基本思想

- バルクがエキスパンションごとに並んでいる前提で、まず探しやすさを優先する。
- 「トークンを出すカード」ではなく「バルクから抜く現物」を主役にする。
- 現行環境に関係ない古いデッキリストは混ぜない。
- 数百デッキ規模で見るが、一度巡回したページはローカルキャッシュして再利用する。
- 最終的な現場判断のために、出すカード・採用デッキ数・採用元も見えるようにする。

### カウンター類はスコープ外

+1/+1カウンター・時間カウンター（Impending）・忠誠カウンターなど、**サイコロで管理するもの**はこのツールでは扱わない。テーブルトップではダイスがあれば足りる。このツールが扱うのは、物理的なカード状のオブジェクト（トークン・紋章・物理的なコピー・裏向き補助など）だけ。

### コピー判定は「盤面に置く現物」だけ

「copy」という単語がOracleに出ていても、すべてをコピー・トークン扱いにはしません。たとえば「呪文のコピーを唱える」「prepared の注釈文で spell の copy が出る」といった処理は、盤面に置くカード状の現物ではないため対象外です。

コピー・マーカーを候補化するのは、`token that's a copy`、`create ... copy`、Offspring など、実際にコピー・トークンやコピー用の現物を用意する必要がある表現に限ります。

### 巡回は「条件が決まってから」

フォーマットと環境期間が決まった段階で、すでにその条件で絞り込まれたURLを起点にクロールする。汎用のデッキリスト一覧（全フォーマット・全期間混在）から始めない。

- 晴れる屋のメタゲームページは `dateFrom=` パラメータ付きのアーキタイプ別URLを持つ → それをそのまま起点にする
- magic.gg / MTGO はフォーマット名がURLに入っているため、フォーマットに一致するリンクだけを辿る
- 記事・カバレージ系は日付フィルタが効かず古いデッキを拾うため除外する

## できること

- Standard / Pioneer / Modern / Legacy を選択
- 大会日を指定
- Standardでは環境開始日を表示
- 直近環境内のデッキリストを巡回
- 巡回先のフォーマット属性を見て、Standard検索にHistoric/Legacy等のデッキが混ざるのを防止
- デッキ内で使われているトークン生成カードを抽出
- トークン、紋章、物理的なコピー、裏向き補助を候補化
- 出来事（Adventure）・前兆（Omen）カードのフェイス名を表示
- 計画（Plot）カードを除外ゾーン追跡マーカーとして候補化
- エンジン始動（Start Your Engines! / DFT）カードを対応
- 昼夜変身（Daybound/Nightbound）マーカーを候補化
- TDR: Endure（Spiritトークン）・Mobilize（Warriorトークン）・Behold（宝物）に対応
- FIN: Job Select（Heroトークン）に対応
- エキスパンション順と種類別を切り替え
- 発売日の昇順/降順で並べ替え
- 採用デッキ数を表示
- デッキ内カードの一致からアーキタイプを推定
- イゼット果敢、イゼット講義、緑単上陸、アゾリウス・モモなど、日本語圏で流通するデッキ名称をアーキタイプとして表示
- アーキタイプごとに色、ゲームプラン、エンジン、トークン準備リスクを補足表示
- メタ傾向を折りたたみ表示の円グラフで表示
- 検索したデッキ/リスト一覧を折りたたみ表示し、初期表示は10件に抑える
- 「抜いた」チェックと未チェックのみ表示
- 印刷用表示
- 巡回ページのローカルキャッシュ
- 検索結果を先に表示し、日本語名・日本語画像などの重い補完は裏で反映
- **カード画像の日英切替**: EN/JA トグルで、取得できた日本語画像だけを切り替え。日本語画像がないものは英語画像で表示
- **日本語名表示**: 発生源カードの日本語テキスト、Scryfall関連カード、トークン種別の辞書、運用ラベルの順で補完し、トークン/マーカーの日本語名欄を空にしない
- **ホバーズーム**: トークン画像にマウスオーバーで拡大プレビューを表示
- **リアルタイムログパネル**: 巡回・Scryfall照合の進捗をページ下部のパネルにリアルタイム表示

## 巡回元

初期URLには以下を入れています。起点はすべてメタゲームページ（条件で絞り込み済み）です。

- 晴れる屋メタゲームページ（フォーマット別: Standard=1, Pioneer=20, Modern=2, Legacy=3）
- MTGTop8（フォーマット別フォーマットページ）
- magic.gg Decklists（フォーマット名がURLに入るイベントだけ辿る）

以下は**除外**しています。

- MTGO公式 (`mtgo.com`) — SPA構成で静的HTMLにデッキデータなし。公開APIも存在しないため
- MTGGoldfish — Cloudflareによるアクセス制限で頻繁にブロックされるため
- 晴れる屋記事 (`article.hareruyamtg.com`) — 記事は日付に関係なく全期間が混在するため
- mtg-jp Coverage — 同上
- 晴れる屋デッキ一覧 (`/ja/deck/`) — 全フォーマット・全期間混在のため

URL欄は手動編集できます。MeleeやSpellBinderなど、イベント固有URLを直接入れる使い方も想定しています。

日本国内の大会準備で使う前提なので、国内ショップ/国内公式記事を優先的に混ぜています。海外メタだけでなく、日本語圏の大会・記事・ショップデッキリストからも拾うことで、国内配信卓で出やすいカードに寄せます。

紙の大会向けツールなので、巡回元からArena専用のデッキリストサイト（MTG Arena Zone）は外しています。トークン発生源の母集団も `game:paper` で絞り、Alchemyなど紙に存在しないカードは候補から除外します。

### サイト別の実態と制限

| サイト | デッキデータ | 備考 |
|---|---|---|
| magic.gg | ✅ 構造化済み | `<deck-list>` タグにカードリスト入り。`format` 属性が検索フォーマットと違うデッキは除外 |
| MTGTop8 | ✅ 構造化済み | HTMLのhrefがクォートなし記法のため専用対応済み。`f=ST` / `PI` / `MO` / `LE` でフォーマットを確認 |
| 晴れる屋（メタページ） | ⚠️ テキストのみ | アーキタイプ名の抽出元として機能。個別デッキページはAWS WAFでブロック |
| MTGO公式 | ❌ JS必須 | SPA構成のため静的HTMLにデッキデータなし。公開APIも見つからないため除外 |

AWS WAFやJS必須ページ（「JavaScript is disabled / JavaScript Required」等）は自動検出してスキップし、巡回失敗・JSブロック件数として別カウントします。カードリスト構造は見つかったが中身をカード名として抽出できなかったデッキは、分類母集団から外し、抽出失敗デッキ数としてサマリーに表示します。サマリーのサイト別チップで各ソースの貢献ページ数・デッキ数を確認できます。

## 環境判定

Standardでは、単純な「直近N日」ではなく、現在のStandard環境が始まった日から大会日までを対象にします。

2026-05-30時点のデフォルトでは、以下の扱いです。

- 2026-04-24: Secrets of Strixhaven発売によりStandard環境が更新
- 2026-05-18: Banned and Restricted AnnouncementはStandard変更なし

そのため、2026-05-30を大会日にした場合は、2026-04-24から2026-05-30までのデッキを対象にします。

## 必要デッキ数の考え方

「絶対に出し漏れがない」ことは、1人だけが使うローグカードまで考えると保証できません。そこでこのツールでは、以下の考え方を使います。

```text
使用率p%以上の発生源を、confidence%以上の確率で少なくとも1回見る
```

必要デッキ数は概算で以下です。

```text
n >= log(1 - confidence) / log(1 - p)
```

目安:

- 5%以上を95%で拾う: 約59デッキ
- 2%以上を95%で拾う: 約149デッキ
- 1%以上を95%で拾う: 約299デッキ
- 0.5%以上を95%で拾う: 約599デッキ

大会規模を選ぶと、この考え方で検索デッキ数を自動提案します。

検索デッキ数の上限は600です。これは、0.5%以上のかなり薄い採用率の発生源でも95%以上の確率で少なくとも1回見るための目安が約599デッキだからです。

配信準備では、600件を超えてさらに薄いローグ採用まで追うより、国内ソースや大会固有URLを増やして母集団の質を上げるほうを優先します。

## メタ傾向

アーキタイプは次の順で決めます。

1. **メタゲームページから自動取得した既知アーキタイプ名との照合**: 晴れる屋のメタゲームページをクロール中にアーキタイプ名一覧を抽出し、デッキタイトルやページタイトルと照合する。手動でMTGGoldfish等のアーキタイプURLを入れた場合も、取得できたアーキタイプ名を照合に使う。
2. **デッキリストにアーキタイプ名（subtitle等）が含まれていればそれを使う**: subtitle は pageTitle と混ぜずに単独でサニタイズして使う（「- Player |」のような誤値は除去）。
3. **日本語圏のデッキ名称辞書との照合**: イゼット果敢、イゼット講義、イゼット・スペルメンタル、セレズニア上陸、アゾリウス・モモなど、国内記事・ショップ・配信で使われる名称を、単なる翻訳ではなくアーキタイプ名として扱う。
4. **カード構成プロファイルとの照合**: アーキタイプ名が付いたデッキから「そのアーキタイプで40%以上のデッキに入っているカード」をコアカードとして学習し、コアカードの25%以上かつ2枚以上が一致したアーキタイプに割り当てる。
5. **残ったデッキを既知アーキタイプ名で再照合**: 上記で `Unknown` のままのデッキ（またはアーキタイプ名が正規でないデッキ）をタイトル・ページタイトルで再度照合する。
6. **土地色＋戦略シグナルによるフォールバック推定**: 上記すべてで未分類のデッキに対して、デッキ内の土地名から色（基本土地・フェッチ・ショック・チェック・ファスト・パスウェイ・トライオーム等）を検出し、既知の打ち消し/全体除去/バーン/1マナアタッカーからコントロール/アグロを判定して「Dimir Midrange」「Mono-Red Aggro」「Azorius Control」のような大枠ラベルを付ける。色も戦略シグナルも得られない場合のみ `Unknown` を維持する。

アーキタイプ名の推定では、サイトのブランド名（晴れる屋・MTGGoldfish等）をタイトルから除去したうえで、Izzet/Dimiといったカラーワードや Aggro/Midrangeといった戦略ワードを含むセグメントを優先して抽出します。「Player」などプレイヤー名由来の文字列は除去します。

どのアーキタイプにも一致しないデッキは `Unknown` として扱いますが、メタ傾向のデッキ数には含めます。カードリストを抽出できたデッキだけを分類母集団にするため、メタ傾向の合計は「抽出成功した検索デッキ数」と一致します。抽出失敗デッキがある場合はサマリーに別表示されます。

推定したアーキタイプは、検索結果のメタ傾向グラフと、各トークンを出すカードの補足表示に使います。メタ傾向は現物準備の主情報ではないため初期状態では折りたたみ、必要なときだけ開けるようにします。アーキタイプIdentityには、表示名、英語名、日本語ローカル名、色、マクロプラン、エンジンタグ、トークン準備リスクタグを持たせ、メタを見るだけで「何のデッキか」と「どの現物確認が危ないか」が分かるようにします。

## キャッシュ

巡回済みページは `.cache/pages` に保存されます。

- `巡回キャッシュを使う`: 保存済みページを優先
- `今回は再取得`: キャッシュを無視して再取得
- `キャッシュ削除`: 保存済み巡回ページを削除

`.cache/` はGit管理しません。

## 開発

サーバー側は `server.mjs`（[Hono](https://hono.dev/) によるHTTPルーティング）と、責務ごとに分割した `lib/` で構成します。

```
npm install   # hono / @hono/node-server のみ
npm start
```

| ファイル | 役割 |
| --- | --- |
| `server.mjs` | Hono ルーティング・静的ファイル配信。検索APIと日本語名/画像の後追い補完API |
| `lib/config.js` | ポート・パス・定数（`maxMatchedCards`・`scryfallCacheTtlMs`・`fetchTimeoutMs`） |
| `lib/data.js` | トークン和名表・巡回元・公式確認済み日本語名/画像上書き表。環境イベントは `data/environment-events.json` を読み込む |
| `data/environment-events.json` | Standard 環境イベント（B&R・ローテーション等）の定義。コード変更なしで更新可能 |
| `lib/util.js` | 汎用ユーティリティ（日付・正規表現・画像URL・セット名） |
| `lib/html.js` | HTMLからのタイトル・日付抽出とテキスト正規化 |
| `lib/cache.js` | ページキャッシュ、`fetchPage` / `fetchJson` |
| `lib/environment.js` | 環境開始日の判定 |
| `lib/archetype.js` | アーキタイプ推定・プロファイル学習・集計 |
| `lib/deck.js` | デッキリストとリンクの抽出 |
| `lib/scryfall.js` | Scryfall API（候補取得・日本語名・関連カード）、日本公式カードギャラリー由来の公式確認済み補完、紋章名生成。in-memoryキャッシュに4時間TTL |
| `lib/tokens.js` | トークン/紋章/コピー等の現物オブジェクト構築。タイプ行からの日本語名補完と運用ラベル生成 |
| `lib/search.js` | カード言及検索とデッキ集計 |
| `lib/crawl.js` | 巡回元のクロール |

依存方向は一方向で循環はありません。

## 注意

- サイト側のHTML構造変更やアクセス制限で巡回できないページがあります。
- Scryfall照合母集団の枚数は、デッキ数に応じて変わる数字ではありません。検索デッキ数で変わるのは、ヒットした生成カード数、現物の種類数、採用デッキ数です。
- 生成物の判定はOracle、Scryfall関連カード情報、公式確認済み日本語テキストに基づきます。配信前の最終確認ではカード画像やOracleも確認してください。
- 日本語画像は取得できたものだけ切り替わります。日本語名は空にしませんが、`名称要確認` を含むものは公式トークン名の追加確認が必要です。
