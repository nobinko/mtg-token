# MTG Token Finder

フィーチャー卓の配信準備で、バルクのトークン束から必要な現物を抜き出すためのローカルWebツールです。

このツールは「カード検索」ではなく「現場の準備表」です。デッキリストを集め、トークン・紋章・物理的なコピー・裏向き補助など、プレイ中にカード状の現物として必要になりそうなものを、探しやすい順に並べます。

## すぐ使う

リポジトリのフォルダにある **`start.bat`** をダブルクリックします。

- 初回だけパッケージを自動インストールします。
- 数秒後にブラウザで `http://localhost:5177` が開きます。
- 使い終わったら黒いウィンドウを閉じます。

PowerShellで起動する場合:

```powershell
npm install
npm start
```

## 現在できること

- Standard / Pioneer / Modern / Legacy を選択
- 大会日から現環境開始日を判定し、古いデッキリストを混ぜずに巡回
- MTGO公式、MTGTop8、magic.gg、晴れる屋メタページなどをフォーマット別に巡回
- デッキ内で使われているトークン生成カードを抽出
- トークン、紋章、物理的なコピー、裏向き補助、計画/昼夜などの現物候補を表示
- 採用デッキ数、採用元、アーキタイプ傾向を表示
- 日本語名と日本語画像を後追い補完
- エキスパンション順/種類別、発売日順、未チェックのみ表示、印刷用表示
- 巡回ログをリアルタイム表示
- 検索デッキ/リスト数を20-600件の範囲で扱う

## 開発

サーバー側は `server.mjs` と `lib/`、画面側は `public/` に分かれています。詳しい責務分担は [docs/architecture.md](docs/architecture.md) を見てください。

```powershell
npm install
npm start
```

## 注意

- サイト側のHTML構造変更やアクセス制限で巡回できないページがあります。
- Scryfall照合母集団の枚数は、デッキ数に応じて変わる数字ではありません。検索デッキ数で変わるのは、ヒットした生成カード数、現物の種類数、採用デッキ数です。
- 生成物の判定はOracle、Scryfall関連カード情報、公式確認済み日本語テキストに基づきます。配信前の最終確認ではカード画像やOracleも確認してください。
- 日本語画像は取得できたものだけ切り替わります。日本語名は空にしませんが、`名称要確認` を含むものは公式トークン名の追加確認が必要です。
