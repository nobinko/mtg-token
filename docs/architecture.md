# 設計

このドキュメントは、MTG Token Finder の判断基準と実装境界をまとめます。使い方は [usage.md](usage.md)、巡回元の実測は [decklist-sources.md](decklist-sources.md)、変更履歴は [changelog.md](changelog.md) を参照してください。

## 設計思想

- バルクがエキスパンションごとに並んでいる前提で、まず探しやすさを優先する。
- 「トークンを出すカード」ではなく「バルクから抜く現物」を主役にする。
- 現行環境に関係ない古いデッキリストは混ぜない。
- フォーマットと大会日が決まったら、公式セット発売日または禁止改定から現環境開始日を先に確定する。
- 検索上限はページ数ではなく、デッキ/リスト数を主基準にする。
- メタ傾向は補助情報であり、現物準備の主情報は「出すカード」「必要現物」「採用元」に置く。

## データフロー

```text
UI入力
  ↓
server.mjs
  ↓
環境開始日を解決
  ↓
Scryfallから候補カード母集団を取得
  ↓
デッキリストソースをクロール
  ↓
デッキカード名を抽出
  ↓
デッキ内の候補カード言及を集計
  ↓
トークン/紋章/コピー/裏向き補助などの現物候補を生成
  ↓
検索結果を返す
  ↓
日本語名/日本語画像を後追い補完
```

## 巡回方針

巡回は「条件が決まってから」行います。汎用のデッキリスト一覧や記事ページから広く掘ると、フォーマット外・環境外の古いデッキが混ざるためです。

- MTGO公式は `/decklists` から `/decklist/{format}-...` 形式のリンクだけを辿り、`window.MTGO.decklists.data` JSONを読む。
- MTGTop8はフォーマットページからイベントページ、個別デッキページへ進む。
- magic.ggはフォーマット名がURLや `<deck-list format>` に現れる時だけ採用する。
- 晴れる屋はメタゲームページをアーキタイプ補助として読む。検索結果 `result` はAWS WAF challengeで本文が返らないため自動巡回しない。
- 記事・カバレージ系は日付フィルタが効かず古いデッキを拾うため、デフォルト巡回元にはしない。

ソースごとの現在の実測結果は [decklist-sources.md](decklist-sources.md) に残します。ソース側の制限は変わるため、デフォルト巡回元を変える前に再測定してください。

## 環境判定

単純な「直近N日」ではなく、現在の各フォーマット環境が始まった日から大会日までを対象にします。環境開始日は `data/environment-events.json` の公式イベント表から解決します。

検索APIは、環境開始日を解決できない場合、古いデッキを混ぜる代わりに検索を実行しません。

## デッキ抽出と失敗の扱い

カードリスト構造は見つかったがカード名を抽出できなかったデッキは、分類母集団から外します。抽出失敗デッキ数はサマリーに別表示します。

これにより、抽出に失敗したページを `Unknown` アーキタイプとして水増ししません。メタ傾向の合計は「カード名を抽出できた検索デッキ数」と一致します。

## アーキタイプ推定

アーキタイプは次の順で決めます。

1. メタゲームページから取得した既知アーキタイプ名と照合する。
2. デッキリストのsubtitleなど、ソース側が持つアーキタイプ名を使う。
3. 日本語圏のデッキ名称辞書と照合する。
4. 正しくラベル付けされたデッキからコアカードを学習し、未分類デッキを再分類する。
5. 残ったデッキを既知アーキタイプ名で再照合する。
6. 土地色と戦略シグナルから大枠ラベルにフォールバックする。

日本語圏で流通するデッキ名は、単なる翻訳ではなく一級のアーキタイプ名として扱います。例: イゼット果敢、イゼット講義、緑単上陸、アゾリウス・モモ。

## 現物候補の判定

候補化するのは、テーブルトップでカード状の現物として置く必要があるものだけです。

- トークン
- 紋章
- 物理的なコピー・トークンやコピー用マーカー
- 裏向き補助
- 計画カードの追跡補助
- 昼夜マーカー

カウンター類はサイコロ管理で足りるため対象外です。

`copy` という単語がOracleに出ても、すべてコピー・トークン扱いにはしません。`token that's a copy`、`create ... copy`、Offspring など、盤面に置く現物が必要な表現だけを対象にします。

## モジュール責務

| ファイル | 役割 |
|---|---|
| `server.mjs` | Honoルーティング、静的ファイル配信、検索API、日本語名/画像の後追い補完API |
| `lib/config.js` | ポート、パス、検索上限、キャッシュTTL、fetchタイムアウト |
| `lib/data.js` | トークン和名表、巡回元、公式確認済み日本語名/画像上書き表、フォーマット定義 |
| `data/environment-events.json` | 環境イベント定義。コード変更なしで更新可能 |
| `lib/environment.js` | 大会日から環境開始日を解決 |
| `lib/cache.js` | ページキャッシュ、HTML/JSONのHTTP取得、HTTPエラー時の期限切れページキャッシュ代替 |
| `lib/crawl.js` | 巡回元のクロール、サイト別統計、JS/WAFブロック検出 |
| `lib/deck.js` | デッキリスト抽出、リンク抽出、フォーマット外リンク除外 |
| `lib/archetype.js` | アーキタイプ辞書、推定、プロファイル学習、集計 |
| `lib/scryfall.js` | Scryfall API、候補カードディスクキャッシュ、関連カード、日本語名、日本語画像、公式補完 |
| `lib/tokens.js` | トークン/紋章/コピー等の現物オブジェクト構築 |
| `lib/search.js` | デッキ内カード言及検索と採用元集計 |
| `lib/html.js` | HTMLタイトル、日付、テキスト正規化 |
| `lib/util.js` | 汎用ユーティリティ |
| `public/app.js` | ブラウザ側UI、検索結果描画、後追い補完反映 |
| `public/styles.css` | 画面と印刷用スタイル |

依存方向はおおむね `server.mjs` から `lib/` へ流し、`lib/` 間の循環を作らない方針です。

## 変更時のチェック

実装を変えたら、最低限次を確認します。

```powershell
node --check server.mjs
node --check public/app.js
node --check lib/deck.js
node --check lib/crawl.js
node --check lib/tokens.js
git diff --check
```

触ったファイルに応じて `node --check` の対象は増減します。巡回元やパーサを変えた場合は、[decklist-sources.md](decklist-sources.md) の再測定も行います。API入力境界を変えた場合は、範囲外・不正値・空値が安全な既定値に落ちることも確認します。
