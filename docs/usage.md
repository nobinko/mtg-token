# 使い方

MTG Token Finder は、配信卓や大会準備で「どのトークン現物をバルクから抜くか」を決めるためのローカルWebツールです。

## 初回起動

### 1. Node.js をインストールする

https://nodejs.org/ja/ を開き、「LTS」と書かれたボタンでインストーラをダウンロードして実行してください。途中の選択肢はデフォルトのままで構いません。

### 2. `start.bat` をダブルクリックする

リポジトリのフォルダにある **`start.bat`** をダブルクリックします。

- 黒いウィンドウが開き、サーバが起動します。
- 初回だけパッケージをインストールします。
- 数秒後にブラウザが自動で開きます。

ブラウザが開かない場合は、手動で次を開きます。

```text
http://localhost:5177
```

### 3. 終了する

黒いウィンドウを閉じてください。サーバも停止します。

## PowerShellで起動する

```powershell
npm install
npm start
```

2回目以降は `npm start` だけで起動できます。停止するときは、サーバを起動したPowerShellで `Ctrl+C` を押します。

## 基本の使い方

1. フォーマットを選びます。
2. 大会日を指定します。
3. 必要な検索デッキ数を選びます。
4. 検索を実行します。
5. 表示された現物候補をバルクから抜き、抜いたものにチェックを入れます。
6. 必要なら印刷用表示に切り替えます。

巡回元URL欄は手動編集できます。イベント固有のURLや、デフォルト巡回元に入れていないサイトの個別URLを試す場合は、1行に1URLずつ追加します。

## このツールが扱うもの

扱うのは、テーブルトップ上でカード状の現物として用意したいものです。

- トークン
- 紋章
- 物理的なコピー・トークンやコピー用マーカー
- 裏向き補助
- 計画カードの追跡補助
- 昼夜マーカー
- Start Your Engines! など、現場で補助物が必要になりやすい処理

## スコープ外

ダイスで管理すれば足りるカウンター類は扱いません。

- +1/+1カウンター
- 時間カウンター
- 忠誠カウンター
- Impending の待機カウンター

`copy` という単語がOracleに出るだけでは、コピー・トークン扱いにしません。「呪文のコピーを唱える」など、盤面に置くカード状の現物が不要な処理は対象外です。

## 検索デッキ数の目安

「絶対に出し漏れがない」ことは、1人だけが使うローグカードまで考えると保証できません。このツールでは、使用率 `p` 以上の発生源を、指定した信頼度で少なくとも1回見るための目安を使います。

```text
n >= log(1 - confidence) / log(1 - p)
```

95%で拾う目安:

| 使用率 | 必要デッキ数 |
|---:|---:|
| 5%以上 | 約59デッキ |
| 2%以上 | 約149デッキ |
| 1%以上 | 約299デッキ |
| 0.5%以上 | 約599デッキ |

検索デッキ数は20-600件の範囲で扱います。薄いローグ採用を古いデッキで水増しするより、国内ソースや大会固有URLを足して母集団の質を上げる方を優先します。

## キャッシュ

巡回済みページは `.cache/pages` に保存されます。

- `巡回キャッシュを使う`: 保存済みページを優先
- `今回は再取得`: キャッシュを無視して再取得
- `キャッシュ削除`: 保存済み巡回ページを削除

Scryfallの候補カード検索は `.cache/scryfall/` に保存します。APIのレート制限や一時障害が起きた場合でも、期限切れキャッシュがあれば代替使用して検索本体を続行します。Scryfall由来データは24時間キャッシュし、検索系APIは低頻度に絞ります。

`.cache/` はGit管理しません。

## ローカル利用と外部サイト

このツールは身内のローカル準備用途の非公式Fan Contentです。WAF、Cloudflare、robots.txt、明示的なアクセス制限を回避して巡回する用途ではありません。巡回できないサイトは、手動で公開デッキURLを足すか、別の公開ソースを使います。

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「Node.js が見つかりません」と表示される | https://nodejs.org/ja/ からインストールして再起動 |
| ブラウザが開かない | 手動で `http://localhost:5177` を開く |
| 「インストールに失敗しました」と表示される | ネット接続を確認して `start.bat` を再実行 |
| 黒いウィンドウが一瞬で閉じる | PowerShellでこのフォルダを開き、`.\start.bat` を実行して表示されたエラーを確認 |
| `npm install` が権限エラーになる | PowerShellを開き直すか、必要なら管理者として実行 |
