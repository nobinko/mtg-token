# 変更履歴

実装の詳細設計は [architecture.md](architecture.md)、巡回元の実測は [decklist-sources.md](decklist-sources.md) を参照してください。

## 2026-06-02

### 監査対応

- 検出した生成カードを現物化する前の上限を100件から600件に引き上げ。
- ログパネルを初期折りたたみにし、モバイルでも下余白を確保。
- 検索ID付きログ配信に変更し、画面では現在の検索ログだけを表示。
- フォーマット不一致の `<deck-list>` を全文フォールバックで拾い直さないよう修正。
- Scryfall検索系APIの呼び出し間隔を500ms以上にし、Scryfall由来キャッシュを24時間保持に変更。
- 非公式Fan Content表記と、アクセス制限を回避しない運用方針をREADME/利用手順/画面に追加。
- カード画像の表示を `contain` にして、著作権表記やアーティスト名を切り落としにくくした。
- Node.js標準テストを追加し、デッキ抽出、リンクフィルタ、検索集計、コピー判定、検出上限を検証。

### ドキュメント体系の整理

- READMEを入口専用に短縮。
- 利用手順を [usage.md](usage.md) に分離。
- 設計思想、データフロー、巡回方針、環境判定、アーキタイプ推定、現物候補判定、モジュール責務を [architecture.md](architecture.md) に分離。
- デッキリスト取得元の実測根拠を [decklist-sources.md](decklist-sources.md) に分離。
- 変更履歴をこのファイルに分離。
- 検索デッキ数の20-600件範囲、手動巡回元URL、フロントエンド変更時の検証手順を各ドキュメントに反映。

### 入力防御と検索ボタン制御

- 検索中に無効化するボタンが画像言語ENボタンになっていたため、検索ボタンを直接参照するよう修正。
- API直叩き時の `maxChildPages` を20-600件に正規化し、不正値は既定値300件に戻すよう修正。

### デッキリストソース実測

- 晴れる屋メタページは取得できるが、検索結果 `result` はAWS WAF challengeで本文0バイトになることを確認。
- 晴れる屋個別 `show` は取得できる例があるが、現行パーサではカード名を抽出できないことを確認。
- MTGO公式は `window.MTGO.decklists.data` JSONからカード名を抽出できることを確認。
- MTGTop8はフォーマットページからイベント/個別デッキへ進み、カード名とイベント日を抽出できることを確認。
- MTGGoldfish個別デッキとMTGDecksはCloudflare 403を確認。

### フォーマット横断の巡回元を再設計

- MTGO公式をデフォルト巡回元に復帰。
- `/decklists` から `/decklist/{format}-...` 形式のリンクだけを辿り、ページ内JSONからリーグ/チャレンジ両方のカード名・公開日を抽出。
- Standard / Pioneer / Modern / Legacy のフォーマット定義を使い、`modern` が `premodern` に誤一致しないようURLセグメントで判定。
- 晴れる屋のメタページはアーキタイプ補助として読むが、AWS WAFで止まる `result` ページは自動巡回しない。
- MTGDecksとMTGGoldfishは完全デッキリストのデフォルト取得元から外す。
- 2026-06-02時点のLegacy環境開始日を2026-05-18として、現環境内デッキはMTGO公式32件 + MTGTop8 171件 = 203件まで取得。299件に届かない場合はソース枯渇として表示する。

### コピー判定とエラー出力

- コピー・マーカーの検出を、盤面に置く現物が必要な表現に限定。
- `Emeritus of Truce // Swords to Plowshares` のように「呪文のコピー」を注釈文で参照するだけのカードをコピー・トークン候補にしないよう修正。
- サーバー例外時に空オブジェクトではなくスタックまたはメッセージをログに出すよう改善。
